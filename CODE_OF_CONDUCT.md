# Code Of Conduct

This project expects professional, respectful collaboration focused on the
technical work.

## Expected Behavior

- Be direct, factual, and constructive.
- Assume good intent while still asking for evidence when claims affect
  correctness, compatibility, security, or performance.
- Keep issue and pull request discussion on topic.
- Respect maintainer decisions about scope, timing, and release readiness.

## Unacceptable Behavior

- Harassment, threats, hate speech, or personal attacks.
- Sustained disruption, spam, or off-topic arguments.
- Publishing private security details before a mitigation is available.
- Repeatedly ignoring maintainer requests to move or narrow a discussion.

## Enforcement

Maintainers may edit or hide comments, close or lock issues, reject pull
requests, or block participants who violate this code. Security-sensitive
conduct issues should be reported privately through GitHub Security Advisories:
<https://github.com/mikediarmid/ycrdt/security/advisories/new>.
