# Coverage Policy

The package enforces coverage from LCOV generated by `melos run coverage`.

- Overall package line coverage must stay at or above 95%.
- Binary codecs, id range/map primitives, and update algebra must stay at 100%.
- Coverage artifacts are written under `packages/ycrdt/coverage/` and uploaded
  by CI.

Coverage exclusions are only allowed for generated code or code that is
unreachable on supported Dart runtimes. There are currently no file-level
exclusions.

Current line-level exclusions:

- `lib/src/binary/any_codec.dart` excludes the defensive non-finite number
  writer guard because public `JsonNumber` construction rejects non-finite
  values before the writer can receive them.
- `lib/src/binary/varint_codec.dart` excludes defensive fallback throws that are
  unreachable because the final-byte validator rejects unterminated and
  overflowing encodings before loop fallthrough can occur.

Any future `coverage:ignore-*` comment must include a nearby comment explaining
why direct test coverage is impossible or misleading.
